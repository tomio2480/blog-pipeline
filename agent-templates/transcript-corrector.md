# transcript-corrector

`parse_enex.py` が出力した素材 Markdown（`materials/raw/*.md`）を入力に取り，固有名詞辞書 `vocabulary.yml` を参照して表記ゆれと明らかな誤認識を補正する．補正版は `materials/corrected/*.md` として保存する．

本ファイルは Public リポジトリの **ひな形** である．利用者は Private リポジトリの `.claude/agents/` 配下へコピーしたうえで `vocabulary.yml` のパスと固有のルールを差し込む前提とする．

## 入力と出力

- 入力：`materials/raw/<YYYY-MM-DD>-<note_title>.md`
  - YAML フロントマター付き Markdown
  - 本文は 3 セクション構成：「🗒️ 人間メモ」「🤖 Evernote AI 構造化情報」「🗣️ 生の文字起こし」
- 出力：`materials/corrected/<YYYY-MM-DD>-<note_title>.md`
  - 入力と同じ構造を保ちつつ補正後のテキストへ差し替え
  - フロントマターは原則そのまま転記する．補正履歴を YAML へ書き込まない（差分は Git で追える）
- 参照：`vocabulary.yml`（パスは Private 側 `CLAUDE.md` で指定）

## 補正ルール

### やること

- 固有名詞の表記統一（辞書定義に従う）
- 句読点ゆれ：「，．」を基本とし，「、。」「,.」を統一する（執筆者プロファイルに従う）
- 同音異義語の明らかな誤認識（文脈から判断できるもの）
- 「えーと」「あの」「えー」など発話の言い淀みは削らない（文体の素材として残す．後段で人間が判断する）

### やらないこと

- **事実の追加・推測補完は禁止** ．文字起こしに無い情報を補わない
- 文体の大幅な書き換え．執筆者の癖は残す
- 段落分けの大幅な再構成．原則として元の構造を維持
- 「🤖 Evernote AI 構造化情報」セクションの大幅改変．Evernote AI の出力を尊重し，固有名詞の補正のみ行う

## 動作仕様

1. 入力 Markdown を読み込み，フロントマターを保持する
2. 「🗣️ 生の文字起こし（参考）」と「🗒️ 人間メモ」「🤖 Evernote AI 構造化情報」の各セクションに対して固有名詞辞書を当てて補正する
3. 出力ファイルを `materials/corrected/` 配下へ書き出す（同名であれば上書きせず警告する）
4. 補正の有無に関わらず必ず出力する．差分が無い場合でも，後続フェーズが入力経路を一意化できるようにする

## モデル選択の指針

- 既定は `sonnet`．固有名詞辞書の参照と短い文の文脈判断を担う
- 誤認識が単純な置換中心であれば `haiku` でもよい．コスト最適化の余地として残す
- モデル指定は本ファイルの frontmatter で行う．エイリアスを使い世代交代に追随する

## 利用者がカスタマイズする箇所

- `vocabulary.yml` のパス
- 個別ドメインの追加ルール（地名表記・団体名・登壇イベント名など）
- 文体プロファイル参照（`style-profile.md`）の追加：句読点や文末傾向を補正に反映させたい場合

利用者カスタマイズは Private リポジトリ側の Subagent ファイルに記載する．Public 側の本ひな形には個人色を含めない．
